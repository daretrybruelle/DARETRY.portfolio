Projet SQL

Ce projet avait pour objectif de concevoir une base de données relationnelle avec SQLite et de mettre en pratique différentes requêtes SQL afin de manipuler et analyser des ensembles de données.

Objectifs

  • Modéliser et structurer une base de données relationnelle.
  • Créer et gérer des tables avec CREATE TABLE et INSERT INTO.
  • Exploiter la base à l’aide de requêtes SQL variées (SELECT, conditions, tris, agrégations).
  • Réaliser quelques interactions avec la base via des programmes en C et Python.

Travail réalisé

La mise en œuvre s’est organisée autour de plusieurs exercices pratiques :

  • Ouverture et description d’une base existante (communes de France).
  • Création d’une base simplifiée (fournisseurs, produits, commandes).
  • Écriture de requêtes pour interroger et exploiter les données, comme :
    • afficher toutes les communes d’un département donné,
    • calculer des valeurs moyennes (population, densité),
    • déterminer la commune la plus vaste,
    • lister les relations entre fournisseurs et produits.
  • Exécution de certaines requêtes via des scripts en C et Python.

Exemple de requête

SELECT nom_commune, superficie
FROM communes
WHERE departement = 'Dordogne'
  AND superficie > 1500
ORDER BY superficie DESC;

Cette requête retourne toutes les communes de Dordogne dont la superficie dépasse 1500 hectares, triées de la plus grande à la plus petite.

Résultats

Ce projet a permis d’appliquer les principes fondamentaux des bases de données relationnelles et de maîtriser les principales requêtes SQL. Il a également montré l’intérêt de l’intégration des bases dans des programmes pour faciliter l’exploitation et l’automatisation des traitements.
